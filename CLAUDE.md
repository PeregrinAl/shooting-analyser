# Заметки для Claude Code

Эти заметки видит ассистент в любой будущей сессии в этом проекте. Держим
коротко — детали и контекст живут в коде и в `memory/`.

## Жанр и приоритеты

Локальный инструмент **дисперсионного анализа** серий стрелковой подготовки.
Принципиальное требование заказчика: **никаких LLM / ML / нейросетей**, только
классические статистические критерии. Статистика выполняется через Pyodide
(`scipy`/`statsmodels`/`numpy`) — нужна сходимость с эталонными реализациями
вплоть до численного значения.

Кросс-платформенность: один и тот же фронт идёт в веб (GitHub Pages) и в
Tauri 2 (Windows/macOS/Linux).

## Структура

- `src/core/` — парсеры входных файлов, статистика, генератор заключения.
  Платформонезависимое, тестируется отдельно.
- `src/ui/` — React-компоненты, общие для веба и десктопа.
- `src/shells/` — `Shell` интерфейс + реализации `web.ts` / `tauri.ts` и
  детектор `detect.ts`. UI работает только через `Shell`, не зная, где он
  запущен.
- `src-tauri/` — Rust-крейт Tauri 2. Разрешения — `capabilities/default.json`.
- `tests/` — Vitest.

TS-алиасы: `@core/*`, `@ui/*`, `@shells/*`.

`src-tauri/icons/*.png` — временные плейсхолдеры (одноцветные), потому что
макрос `tauri::generate_context!` читает иконки во время компиляции.
Перед первым релизом заменить настоящим логотипом через `tauri icon` —
см. `src-tauri/icons/README.md`.

## Два трека входных данных

1. **Экспорт SCATT Expert** (HTML/текст) — основа v1. Парсер — день работы,
   сразу даёт R, T, 10.0, 10+, скорости траектории, расстояние СТП↔пробоина.
2. **Прямое чтение `.scatt-expert`** (SQLite v5) — отдельная ветка
   `feature/trace-parser`, основа v2. BLOB трасс не сжат и не делится ровно
   на разумный размер отсчёта — вероятно разностное кодирование. Исследовать
   на маленьком файле `storage/scatt-test-10-units.scatt-expert`.

## S1/S2 — это скорости

ТЗ называет S1/S2 «длинами траектории» — заказчик подтвердил, что имелась в
виду **скорость**. Берём как есть из SCATT: «средняя скорость траектории»
и «скорость за 250 мс» (обе мм/с). Длину из скоростей **не** считаем.

## Юридические рамки

- В названии/домене/логотипе не использовать слово SCATT и иные товарные знаки.
- Не коммитить код или ресурсы SCATT Expert.
- Не коммитить чужие пользовательские файлы. `storage/` — в `.gitignore`.

## Стиль коммитов

Коммиты делает **пользователь сама**, каждый проходит через её ревью.
Ассистенту: `git commit`/`git push` не запускать; готовить изменения в рабочем
дереве и отчитываться. `git status`/`git diff` — можно.
